styles.css

The authoritative reference for this app's look and feel. It links the live ./styles.css and renders real components with their real class names, so whatever ships is exactly what you see here. Use it two ways: (1) read the tokens, type scale and spacing below before inventing new values — reuse beats adding; (2) copy a component's markup straight from this page as the scaffold for a new feature or a visual prototype, then restyle only what genuinely needs to differ. Designed and checked at 412 px portrait (Pixel 8a installed PWA) — the only real deployment target.

Design principles

Non-negotiables that every screen inherits. These shape tone and layout as much as the tokens do.

One user, one phoneBuilt solely for Andy on a Pixel 8a. No accounts, onboarding, empty-state copy for strangers, or multi-device logic. Hard-coded personal assumptions are preferred over configurability.
Touch-first, thumb-reachTap targets ≥ 44 px. No hover-only affordances, no right-click menus. Primary actions sit low on the screen.
Humane, not punitiveProgress UX encourages rather than scolds. Avoid alarm-red and brittle single-day swings. Over-target reads as "today went over", never "failure".
Words of affirmationPair numbers with warm, encouraging copy (see .cal-affirm) — never bare figures alone.
Accurate, never inflatedCalorie estimates are unbiased best-guesses. Never pad "to be safe" — inflated numbers cause real distress.
Numbers are tabularAnything numeric uses font-variant-numeric: tabular-nums so digits don't jitter as they change.
Offline is normalFlaky network is the default assumption; anything touching sync must degrade gracefully.

Colour tokens

Every colour is an HSL custom property defined once in :root (light) and overridden in a prefers-color-scheme: dark block. Reference tokens by variable — never hard-code a hex. Swatches below are read live; left chip = light, right chip = dark.

Surfaces & text

Accent & semantic

Day-type tints

Time-of-day categories

Each meal entry gets a soft pastel wash (the token) plus a stronger same-hue 3 px left edge (hard-coded in .entry.category-*::before).

Non-token accents to know about. A few intentional one-offs live outside the token set: the day-context "note" uses a rose rule (hsl(342 52% 52%) light / hsl(342 48% 60%) dark) with a darker eyebrow ink — rose appears nowhere else, so it reads as "special". AI-confidence colours (.ai-status-log.confidence-*) and the entry left-edge hues are also literal HSL. If you find yourself reaching for a new accent, check these first.

Typography

Two families. Fraunces (--font-display, a variable serif with optical sizing) carries every heading, the brand, big numbers and the day-context note — it gives the app its warm, journal-like character. Everything else is the system sans (--font). A monospace (--font-mono) is reserved for raw/technical text (diet profile, sync log). Set font-variation-settings: "opsz" N on Fraunces to match the optical size to the rendered px.

Elevation

Shadows are built from two tokens — --color-shadow and --color-shadow-lg — kept deliberately soft. The hero tiles layer a tight contact shadow with a wide, very faint ambient one. Don't introduce new shadow recipes; reuse a level below.

Iconography

UI chrome icons are inline SVG, line style: fill:none, stroke:currentColor, stroke-width:2 (2.2 for the chevrons), round caps and joins, on a 24×24 viewBox rendered at 20–22 px. They inherit --color-text-muted in icon buttons. Emoji (✨ AI, 📷 camera) are used as lightweight action glyphs inside inputs — not for chrome.

Header & date navigation

App header (brand + icon actions) and the date pager. The pager differentiates day type with a left accent rail and tint: today is brightest (accent rail + pip), yesterday fainter, past neutral. Future days are reached with the right chevron; on today it's disabled.

Day context note

An optional one-line annotation for a day (e.g. "giving blood", "travelling"). Styled as a journal margin note: Fraunces italic, inset from the left with a rose rule and an uppercase eyebrow (both injected via CSS ::before). Tapping it opens the edit overlay.

Entry form & inputs

The Food form expands to: free-text item → calories field with ✨ (AI estimate) and 📷 (photo) → time-category pills → Save. Pills use real radios so the checked wash comes from :has(.category-radio:checked). On today, picking a future time category turns the pill dashed (.category-pill-future) — it plans rather than logs. Measurements use a labelled row with a unit suffix.

AI estimate feedback

After an estimate the .ai-status-log reports confidence with a colour band (excellent / moderate / low / error), or a .match-chip offers a remembered local match to one-tap reuse. Keep this copy factual and calm.

Entries list

Food entries are grouped by time-of-day under .category-header rules. Each .entry carries its category wash + left edge. Planned (future) items render as a dashed, muted draft (.entry-pending). Weight/waist entries show a numeric .entry-value instead of calories.

Calorie tile — all states

The day's hero. A progress ring + headline number + status + detail, with an affirming line. Ring stroke follows the rolling week (so a single big day never flashes failure), while the centre number's colour reflects today only — green on a deficit, red on a surplus. States: is-under, is-over, is-near (within ±150), is-muted (no maintenance estimate yet). Planned food extends a faded arc (.cal-ring-pending).

Weekly pace tile

On the Progress view: how the week is tracking against the weekly goal. Bands map to encouraging language and colour — is-great (glows), is-steady, is-holding, is-gaining, is-muted. Never alarmist.

Buttons & pills

Primary (gradient, the one call to action), ghost (secondary), danger (destructive, soft until hover). All ≥ 48 px tall and press with a subtle scale. Range chips and category pills round out the interactive set.

Settings & panels

Full-screen overlays share .settings-panel chrome: sticky header with a title and a primary "Save & Close", then stacked sections with uppercase labels, inputs, a mono textarea for profiles, and a hint line. Sync status uses .sync-status.is-ok / .is-error / .is-info.

Progress charts

Hand-built SVG on a 360×190 canvas. Weight: faint raw line (.chart-line-weight) under a bold rolling-average trend (.chart-line-avg) with a soft area fill and end dots. Net-balance bars are green under target / amber-red over (.chart-bar.is-under / .is-over / .is-neutral / .is-current), around a zero line. Axis text is .chart-axis-label; the dashed .chart-target-line marks the goal.

Empty state & range chips

Quiet, never naggy. The empty state is a single muted line. Range chips select chart windows (7d / 30d / 90d / All).
