`STATUS.md` at the repo root is the live brain for this project. It holds: current phase, last completed work, spike learnings worth remembering, the prioritised next-step list, phase acceptance criteria, and known follow-ups.

**How to apply:** Read it once at session start to orient. Update it at the end of any working session that changed state. Prune it as phases close — when a phase ends, move the spike-learnings detail out (or delete) so the file stays scannable.
